# Forest Garden — Project Vision

A decentralized collaboration stack enabling coherent multi-agent work at fractal scales — from personal agents through P2P, organizational, and network-of-networks collaboration.

## What This Is

Forest Garden is an agentic commons where human and AI agents collaborate through a four-layer architecture:

1. **Pattern Languages & Protocols** — define how collaboration should work
2. **AD4M / Agent-Centric Semantic Spaces** — define how sovereign agents express meaning, identity, and shared spaces
3. **KOI / Knowledge Operations Infrastructure** — implement knowledge, retrieval, provenance, federation, and governance
4. **Spec DAG / Project Governance** — the governance backbone (vision → architecture → specs → roadmap → code)

## Naming Stack

- **System**: Forest Garden — a self-sustaining knowledge commons
- **Protocol**: Mycorrhizal Federation — sovereign nodes connected through mutualistic exchange
- **Architecture**: Mycelial Holarchy — trees (governance/nesting) + mycelia (knowledge flow/meshwork)

## Dual-Axis Architecture

The system has two coexisting structural axes:

- **Holarchic axis** (trees): Clean nesting for governance, spec DAGs, sovereignty boundaries. Fractal scale: personal → P2P → organizational → network-of-networks → global.  
- **Mycelial axis** (mycelia): Overlapping mesh for knowledge flow, agent membership across projects, lateral federation. Hypergraph structure.

A forest is both trees AND mycelia — neither alone captures the full system.

## Design Principles

- Every schema/API/protocol decision should be clean enough to extract into a standalone package
- No personal-workflow-specific assumptions baked into the core
- Build for plural, decentralized network-of-networks from the start
- Solo developer — tooling must be lightweight, not bureaucratic
